# UNI Katalon Web Automation Framework 🌐
A comprehensive demonstration of Web UI Test Automation using **Katalon Studio**. This repository serves as a foundational example of organizing Web UI element locators, designing scalable test cases, and structuring data-driven suites.

## 🎯 Objective
To provide a clean, maintainable architecture for Web automation projects, emphasizing best practices like the Page Object Model (POM), externalized test data, and robust failure handling.

## 🏗 Architecture & Technologies
- **Test Engine**: Katalon Studio (Selenium WebDriver wrapper)
- **Target**: Web Browsers (Chrome, Firefox, Edge, Safari)
- **Pattern**: Keyword-Driven & Data-Driven Testing

## ⚙️ Key Concepts Demonstrated
1. **Object Repository**: Centralized management of web elements (XPaths, CSS Selectors, IDs).
2. **Custom Keywords**: Reusable utility functions for complex web interactions.
3. **Data-Driven Testing (DDT)**: Parameterizing test cases with external data sources (CSV/Excel).
4. **Test Suites & Collections**: Organizing tests by modules or execution environments.

## 🚀 Getting Started
1. Clone this repository.
2. Open the project folder in **Katalon Studio**.
3. Select an active browser from the execution toolbar.
4. Run the `Test Suites/Web UI Tests` suite and review the generated reports.
